# mod_datalynx — End-User Documentation

Welcome to the end-user documentation for the **Datalynx** Moodle activity.

This documentation is written for:

- **Teachers**
- **Non-editing Teachers**
- **Course Managers**

## Suggested learning path

1. Start with **Getting Started**.
2. Design your data model in **Fields**.
3. Build user-facing layouts in **Views**.
4. Add automation in **Rules**.
5. Define daily operation in **Managing Entries**.
6. Finalize governance in **Permissions**.
7. Optimize UX with **Patterns, Styling, and Tools**.

> **Pro-Tip**  
> Build a first version with minimal complexity, test it with real users, then add advanced patterns and styling.
